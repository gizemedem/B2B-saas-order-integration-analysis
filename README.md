# B2B SaaS Order & ERP Integration Project: Business Analysis & SQL Data Validation

This repository demonstrates an end-to-end business analysis, documentation, and data validation framework for a B2B SaaS E-Commerce platform integrated with an Enterprise Resource Planning (ERP) system.

## 📌 Project Overview
In B2B SaaS solutions, seamless data synchronization between the e-commerce platform and corporate clients' ERP systems is critical. This project focuses on analyzing the "Order & Stock Sync" integration, documenting requirements, writing test cases, and using SQL for data validation and anomaly detection (e.g., price mismatches, stock discrepancies).

## 🛠️ Skills & Tools Demonstrated
- **Business Analysis:** Requirements Gathering, User Stories (JIRA Format), Process Mapping.
- **Data Validation & Analytics:** SQL Server (SSMS), Advanced Querying (JOINs, GROUP BY, CTEs), Data Integrity Checks.
- **Integration & Testing:** API/JSON Structures, UAT Test Cases, Defect Tracking.
- **Tools:** GitHub, Markdown, SQL Server, Excel.

## 📂 Repository Structure
*   `📁 documentation/`: User Stories, Functional Requirements, API Mapping.
*   `📁 sql-scripts/`: Queries for checking stock, price, and order data consistency.
*   `📁 testing/`: User Acceptance Testing (UAT) scenarios and test cases.
