# Stakeholder Analysis – B2B SaaS Integration Project

## 🎯 Internal Stakeholders

### 1. Business Analyst (You)
- Requirement gathering
- Process analysis (AS-IS / TO-BE)
- SQL validation
- Test scenario design

---

### 2. Product Owner
- Prioritizes business needs
- Approves requirements
- Aligns roadmap with business goals

---

### 3. Development Team
- API development
- ERP integration
- Bug fixing and system improvements

---

### 4. QA / Test Team
- Test case execution
- UAT support
- Regression testing

---

## 🎯 External Stakeholders

### 1. Corporate Customers
- Place orders via B2B platform
- Expect accurate pricing and stock info

---

### 2. ERP Systems (SAP / Logo / Netsis)
- Receive order data
- Update stock and financial records

---

## ⚠️ Key Stakeholder Pain Points

- ERP sync delays
- Data inconsistency
- Lack of real-time validation
